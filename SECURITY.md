# Security Policy

KGAID is methodology documentation, but its repository, automation, and release artifacts can still contain security-relevant defects.

Please report suspected vulnerabilities privately to the repository owner through GitHub's private vulnerability reporting feature, or contact **Krzysztof Olejnik** through the GitHub account associated with this repository. Include affected file or workflow, impact, reproduction steps, and any safe mitigation. Do not publish proof-of-concept exploits in an issue before a coordinated response.

The Maintainer will acknowledge a report, assess it, and coordinate a correction and disclosure timeline proportionate to impact. The supported baseline is the latest published KGAID baseline; `KGAID-0.1.0` is prepared but not yet published.
